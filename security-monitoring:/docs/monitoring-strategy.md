## Amazon GuardDuty

GuardDuty was enabled to provide continuous threat detection across AWS resources.

The service analyzes CloudTrail events, DNS logs, and network activity to identify suspicious behavior and potential compromises.

## AWS Security Hub

Security Hub was enabled to centralize security findings and provide visibility into AWS security posture.

The AWS Foundational Security Best Practices standard was enabled to evaluate the environment against AWS security recommendations.

## Monitoring Objectives

The MedSecure Health environment uses AWS-native monitoring services to provide visibility into account activity and security posture.

### GuardDuty

Provides threat detection and behavioral analysis.

### Security Hub

Centralizes findings and security posture assessments.

### CloudTrail

Provides audit logging for API and console activity.

### CloudWatch

Provides operational monitoring and alerting capabilities.
